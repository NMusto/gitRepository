Areas de GIT

Working Directory

Es el área de trabajo donde realizamos todas las modificaciones en nuestro proyecto y corresponde a la carpeta donde se inicializó el mismo.

Staging Area (INDEX)

Corresponde al área de control de cambios, donde se identifican los archivos a los que les vamos a hacer seguimiento y sobre los cuales se va a realizar el snapshot del código. Es importante tener en cuenta que para realizar el commit deben haberse agregado al INDEX por lo menos un archivo y todos aquellos que no figuren.

Repository

Lugar donde se almacenan los cambios identificados mediante las capturas de código en formato BLOB.
